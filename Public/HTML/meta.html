Тег <meta>

Описание

Тег <meta> определяет метатеги, которые используются для хранения информации предназначенной для браузеров и поисковых систем. Например, механизмы поисковых систем обращаются к метатегам для получения описания сайта, ключевых слов и других данных. Разрешается использовать более чем один метатег, все они размещаются в контейнере <head>. Как правило, атрибуты любого метатега сводятся к парам «имя=значение», которые определяются ключевыми словами content, name или http-equiv.

Атрибуты

charset <meta charset="кодировка"> Пример:

<meta charset="utf-8">

Указывает кодировку документа. Атрибут введен в HTML5 и предназначен для сокращения формы тега <meta>, которая задавала кодировку в предыдущих версиях HTML и XHTML.
content <meta content="..."> Пример:

<meta name="keywords" content="HTML, META, метатег, тег, поисковая система">

Атрибут content устанавливает значение атрибута, заданного с помощью name или http-equiv. Атрибут content может содержать более одного значения, в этом случае они разделяются запятыми или точкой с запятой.
http-equiv <meta http-equiv="..."> Пример:

<meta http-equiv="Content-Type" content="text/html; charset=utf-8">

Браузеры преобразовывают значение атрибута http-equiv, заданное с помощью content, в формат заголовка ответа HTTP и обрабатывают их, как будто они прибыли непосредственно от сервера.

Content-Type
Тип кодировки документа.
expires
Устанавливает дату и время, после которой информация в документе будет считаться устаревшей.
pragma
Способ кэширования документа.
refresh
Загрузить другой документ в текущее окно браузера.
name <meta name="..."> Пример:

<meta name="keywords" content="HTML, META, метатег, тег, поисковая система">

Устанавливает идентификатор метатега для пары «имя=значение». Одновременно использовать атрибуты name и http-equiv не допускается.

author
Имя автора документа.
description
Описание текущего документа.
keywords
Список ключевых слов, встречающихся на странице.
